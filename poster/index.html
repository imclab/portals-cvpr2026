Portals: Persistent, Editable 4D Spatial World Models on Edge Devices

1H3M, Inc., USA    2The IMC Lab, USA    imclab.github.io/portals-cvpr2026

1 Contribution

We present Portals, a deployed 4D spatial world-model system for the edge-device era. The paper's core result is not a single isolated representation, but a deployment stack that proves three coupled requirements can be satisfied together: real-time runtime performance on commodity mobile hardware, persistent scene state across revisits, and creator-facing world editing that bridges reconstruction and generation.

  • Edge runtime: SPAG — Spherical Pixel-Aligned Gaussians — w/ LOD-adaptive quality + a shared spatial-media compute substrate that fuses depth, stencil, audio, and ML-pose channels, driving 360+ source-agnostic VFX at 60 fps on iPhone 14 Pro (2.7–4.1× speed-up).
  • Persistence: geospatial scene-state architecture w/ layered world metadata, reloadable scene payloads, and anchor-guided re-alignment across sessions.
  • Creator interface: composition pipeline bridging reconstruction and generation through VFX composition, voice-driven semantic actions (on-device parser + cloud fallback), and no-code authoring.
  • Benchmark axes for production 4D world models — mobile efficiency, geospatial alignment, persistence fidelity, creator editability.

2 Background

3D Gaussian Splatting [Kerbl 2023] became the de facto backbone for AR/VR via real-time rasterization. 4D representations followed: 4D-GS [Wu 2024], K-Planes [Fridovich-Keil 2023], MoSca [Lei 2024], Shape-of-Motion [Wang 2025], GHG [Kwon 2024]. Recent feed-forward systems (CUT3R [Wang 2025], VGGT [Wang 2025], MegaSaM [Li 2025]) trend toward real-time perception. Generation-first models (Genie, WorldGen [Sarafianos 2025]) provide a complementary endpoint. Portals targets the open deployment problem: representations are mature; making them persistent, editable, and authorable on edge devices is not.

6 Shared compute substrate

One depth-to-world compute dispatch per frame (32×32 thread groups · Metal/Vulkan). Outputs bound through a 5-property contract (ColorMap · DepthMap · RayParams · InverseView · DepthRange) consumed by 360+ VFX. O(1) compute scaling — additional effects only add binding + rendering cost, never the shared depth-to-world pass.

7 Geospatial persistence

Three-tier path matched to edge constraints:

  1. Metadata — Firestore-indexed discovery, <300 ms reload
  2. Manifest (XRAI) — scene structure + edit history; out-of-line geometry refs
  3. Payload — .splat / SPZ from object storage, <1 s on LTE

Recovery (in development): GPS coarse (±3–10 m) → ARKit VIO (cm) → photometric alignment vs pre-baked reference splat Lalign = ‖Iref − Ilive‖₁ + λSSIM LSSIM, λSSIM = 0.2.

11 Discussion

Reconstruction vs capture burden. Even w/ ReconFusion-style priors, casual phone video introduces blur and dynamic-object artifacts. Shape-of-Motion [Wang 2025] and MoSca [Lei 2024] improve robustness; real-time monocular 4D capture for in-the-wild AR remains open.

Editability. Compositing a Gaussian avatar into a Gaussian background w/o seams is open. Feature-field extensions [LangSplat] begin to help but do not resolve.

Privacy. Persisting 3D scans at scale raises bystander + interior consent issues. Portals enforces client-side ephemeral buffers + keyframe-only upload. Principled framework remains open.

Limits of this paper. Axis 2 (geospatial alignment accuracy) not yet measured at device scale. Voice eval is automated regression, not user study. Three-signal recovery + photometric relocalization in active development. L3DG-style latent generation = future.

12 Benchmark axes (proposed)

  1. Mobile render efficiency — median + p95 frame time on commodity mobile GPUs
  2. Geospatial alignment accuracy — registration error after re-entry
  3. Temporal persistence fidelity — anchor recovery rate · edit survival · reload cost
  4. Creator editability — semantic-action success · edit-to-preview latency
