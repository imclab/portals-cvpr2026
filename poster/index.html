01

Why persistent 4D?

AR's defining shift is not 3D → 4D. It is stateless → stateful: scenes that persist, compound, and stay editable across sessions, devices, and users.

Recent 4D representations (4D-GS, K-Planes, MoSca, Shape-of-Motion, GHG) solve dynamic-scene modeling in the lab. The deployment problem is open: real-time on a phone, persistent across visits, authorable by non-experts.

Goal Make 4D worlds live on edge devices — render in real time, survive revisits, stay editable through voice / gesture / no-code.
  • Clinical AR · SPECT/CT on Magic Leap (MSK · NIH/NCI)
  • Live events · concerts · motion-capture stages
  • Consumer · geospatially anchored scenes

04

Creator interfaces · deployed

Voice + gesture + no-code, lowering to the same scene-graph operations. Three steps: photo → upload → publish.

Benchmark axes for production 4D world models

  1. Mobile render efficiency · median + p95 frame time on commodity mobile GPUs
  2. Geospatial alignment accuracy · registration error after session re-entry
  3. Temporal persistence fidelity · anchor recovery · edit survival · reload cost
  4. Creator editability · semantic-action success rate · edit-to-preview latency

What this paper does not claim

  • Axis 2 (geospatial accuracy) — not yet measured at device scale
  • Voice eval — automated regression tests, not a user study
  • Three-signal recovery + photometric relocalization — in active development
  • L3DG-style latent Gaussian generation — future work
